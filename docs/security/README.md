# فهرس وثائق الأمان — Wardah ERP

هذا المجلد يحتوي وثائق من مراحل زمنية مختلفة. لا تُقرأ جميعها باعتبارها وصفًا
للحالة الحالية.

## ترتيب القراءة المعتمد

1. **`REMAINING_SECURITY_FIXES.md`**  
   السجل الحالي المعتمد للحالة الحية، المخاطر المفتوحة، الأولويات، وحالة النشر.

2. **`SECURITY_MODEL.md`**  
   النموذج الأمني المستهدف والمبادئ المعمارية. بعض البنود فيه طموحة أو غير متحقق
   منها على الإنتاج، لذلك لا يستخدم منفردًا كدليل امتثال.

3. **`PERMISSIONS_MAP.md`**  
   مسودة خريطة الصلاحيات والأدوار. تحتاج مواءمة مع Issue #48، وقاعدة البيانات
   الحالية، وصلاحيات الأعمال الفعلية قبل اعتمادها.

4. **Runbooks ونتائج migrations**  
   تستخدم لإثبات تغيير محدد فقط، ولا تثبت سلامة النظام بالكامل.

## تصنيف الوثائق

| النوع | المعنى |

## وثائق تاريخية

الملفات التالية أو ما يماثلها تبقى للأثر التاريخي، ولا تعتمد لتقرير الحالة
الحالية دون إعادة تحقق:

- `SECURITY_FIXES_SUMMARY.md`
- `SECURITY_CURRENT_STATUS.md`
- `MIGRATION_66_SUCCESS_SUMMARY.md`
- أي وثيقة تعلن `0 warnings` أو `مكتمل بالكامل` بتاريخ قديم

العبارة «0 Security Errors» في migration أو فحص قديم تعني أن **نطاق ذلك الفحص**
نجح وقتها، ولا تعني غياب مخاطر جديدة أو إعدادات منصة أو منح واسعة أو APIs لم
تكن ضمن الفحص.

## قاعدة التحديث

عند تحديث الحالة الأمنية:

1. اجمع دليلًا من البيئة الحية أو CI أو كتالوج PostgreSQL.
2. اذكر التاريخ والبيئة وآخر migration مطبقة.
3. حدّث `REMAINING_SECURITY_FIXES.md` أولًا.
4. لا تغيّر وصف وثيقة تاريخية لإظهارها وكأنها كانت خاطئة في وقتها؛ صنفها فقط.
5. لا تغلق بندًا بسبب إخفاء زر في React؛ يجب إثبات الحارس من الخادم.
6. لا تعتبر تحذير Advisor ثغرة مؤكدة أو تتجاهله تلقائيًا؛ صنفه مع قرار موثق.
